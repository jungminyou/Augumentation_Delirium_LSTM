# Delirium Prediction Pipeline (SCI Submission Code)

This repository is a modularized version of the original notebook-based workflow used in the SCI manuscript.

## Project Goals

- Reproduce patient-level data split without leakage.
- Generate synthetic minority trajectories using TimeGAN and DTW-SMOTE.
- Train baseline and augmented multi-task models.
- Evaluate discrimination, calibration, and decision utility.
- Compare fidelity between original and synthetic datasets.

## Repository Structure

- `src/config.py`: Shared configuration and paths.
- `src/augmentation.py`: TimeGAN + DTW-SMOTE data augmentation pipeline.
- `src/tensor_prep.py`: 3D tensor conversion and scaling.
- `src/models.py`: Soft-gated multi-task LSTM model definition.
- `src/train.py`: Model training and prediction extraction.
- `src/evaluate.py`: AUROC/AUPRC/calibration/DCA metrics and plotting.
- `src/fidelity.py`: Statistical tests, t-SNE, DTW/Wasserstein, JSD analysis.
- `run_all.py`: End-to-end runner.
- `docs/PIPELINE.md`: Step-by-step scientific workflow documentation.
- `docs/GITHUB_RELEASE_CHECKLIST.md`: Release and anonymization checklist.
- `docs/MANUSCRIPT_CODE_MAPPING.md`: Journal manuscript section to code mapping.

## Quick Start

1. Create and activate a Python environment.
2. Install dependencies:
   - `pip install -r requirements.txt`
3. Update paths in `src/config.py` if needed.
4. Run full pipeline:
   - `python run_all.py`

## Stage-by-Stage Execution

1. Stage 1 (augmentation and split): `python run_stage_01_augmentation.py`
2. Stage 2 (tensor conversion and scaling): `python run_stage_02_tensor_prep.py`
3. Stage 3 (training and evaluation): `python run_stage_03_train_eval.py`
4. Stage 4 (fidelity and statistical analysis): `python run_stage_04_fidelity.py`

## Reproducibility Notes

- All patient-level split operations use fixed random seeds.
- Leakage-prone features (for example `CAM_diagnosis`) are removed before modeling.
- Scaling is fit only on original training data, then applied to validation/test and synthetic data.
